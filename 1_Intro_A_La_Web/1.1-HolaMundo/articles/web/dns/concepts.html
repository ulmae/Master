What is DNS?

DNS stands for Domain Name System. This system is a database that translates a web address into the physical IP address of the computer hosting that site. In that sense, it will serve as a phonebook, relating domains with their respective address. It also serves to control the settings of your domain through DNS records.

DNS Records

DNS records are basically mapping files that tell the DNS server which IP address each domain is associated with, and how to handle requests sent to each domain.


When someone visits a website, a request is sent to the DNS server and the forwarded to the web server provided by a web hosting company, which contains the data from the website. Then, DNS records will tell the web server how to respond to your input.


The types of information elements in your DNS are categorized and organized with a list of DNS record types, the resource records. Each record has a type, an expiration time, a class, and type-specific data. The time that changes to your DNS records take to propagate through the Internet will depend on the expiration time, also called Time To Live (TTL). You can modify these values through your domain host.

DNS record types

There's a number of different DNS records, but the most commonly used are:


A

Also known as an Address record, or host record. It returns a 32 bit IPv4 Address of the computer hosting that domain's services or web host.

AAAA

An AAAA record is similar to the A record, but it allows you to point the domain to an IPv6 address, which is the most recent version of the Internet Protocol.

MX

Mail Exchange records direct a domain's email to the servers hosting the domain's user accounts. Multiple MX records can be defined for a domain, each with a different priority for redundancy purposes. When multiple MX records are defined, if mail can't be delivered using the highest priority record, the second priority record is used, and so on.


Server redundancy defines the ability of a computing infrastructure to provide additional servers that may be deployed on runtime for backup, load balancing or halting a primary server for maintenance purposes.
TXT

A text record provides text information to sources outside your domain that can be used for a number of arbitrary purposes. The record's value can either be human or machine-readable text.

NS

Name server records determine which servers will communicate DNS information for a domain, so they should point to your domain host. A name server is a specialized server on the Internet that handles queries or questions from your local computer, about the location of a domain name's various services.

CNAME

A Canonical Name record links an alias name to another true or canonical domain name. They are stored with a pair of values: the alias you're creating the record for and the domain the alias should point to.


In this way, CNAME records make it easy to run multiple services from one IP Address with the use of subdomains.

DNS Propagation

When you update your name servers for a domain, it may take up to 24 to 72 hours for the change to take effect. This period is called DNS propagation.


In other words, it is a period of time that Internet service provider (ISP) nodes across the world take to update their caches with the new DNS information of your domain.


Due to DNS caches of different levels, after the name servers change, some of your visitor might still be redirected to your old server for some time, whereas other can see the website or service from the new server shortly after the change. Each of the ISP nodes checks its own cache to see if it contains the DNS information of the domain. If it is not there, it looks it up and saves it in order to speed up the loading next time and to reduce the traffic.


That is why the new name servers will not propagate immediately – ISPs have different cache refreshing intervals, so some of them will still have the old DNS information in memory.
